# Инструкция по использованию системы прогнозирования успеваемости

## Запуск приложения

### 1. Запуск сервера
```bash
# В корневой папке проекта
npm start
```
Сервер будет доступен по адресу: http://localhost:5000

### 2. Запуск клиентской части
```bash
# В папке client
npm start
```
Клиент будет доступен по адресу: http://localhost:3000

## Использование системы

### Регистрация и вход

1. **Регистрация нового пользователя:**
   - Перейдите на страницу регистрации
   - Заполните форму: ФИО, имя пользователя, пароль, выберите роль
   - Роли: "student" (студент) или "teacher" (преподаватель)

2. **Вход в систему:**
   - Введите имя пользователя и пароль
   - После успешного входа вы будете перенаправлены на дашборд

### Функции для преподавателей

#### Управление студентами
- **Просмотр списка студентов:** На странице "Студенты" отображается список всех студентов
- **Добавление студента:** Нажмите "Добавить студента" и заполните форму
- **Редактирование данных:** Нажмите на иконку редактирования рядом со студентом
- **Удаление студента:** Нажмите на иконку удаления (осторожно!)

#### Управление оценками
- **Добавление оценок:** На странице студента можно добавить оценки по экзаменам
- **Добавление результатов зачетов:** Можно отметить сданные/несданные зачеты
- **Редактирование:** Все оценки можно изменять

#### Прогнозирование
- **Расчет прогноза:** На странице студента нажмите "Рассчитать прогноз"
- **Просмотр всех прогнозов:** На странице "Прогнозы" отображается статистика
- **Фильтрация:** Можно фильтровать по группам и баллам

### Функции для студентов

#### Просмотр данных
- **Мои результаты:** Просмотр своих оценок и зачетов
- **Мой прогноз:** Просмотр прогноза успеваемости
- **Профиль:** Управление личными данными

## Алгоритм прогнозирования

Система использует следующие параметры для расчета прогноза:

1. **Средняя оценка по экзаменам** (2-5 баллов)
2. **Процент сданных зачетов** (0-100%)
3. **Общий показатель успеваемости** (0-100 баллов)

Формула расчета:
- Нормализация экзамена: `(средняя_оценка - 2) / 3 * 100`
- Взвешенная формула: `70% * экзамены + 30% * зачеты`

## Безопасность

- Все пароли хешируются с использованием bcrypt
- JWT токены для аутентификации
- Разграничение доступа по ролям
- Валидация всех входных данных

## Техническая поддержка

При возникновении проблем:
1. Проверьте логи сервера в консоли
2. Убедитесь, что база данных PostgreSQL запущена
3. Проверьте подключение к базе данных
4. Убедитесь, что все зависимости установлены
